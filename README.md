# Lurix Pro

**Practice Without Friction.**

Lurix Pro is a premium browser extension designed to transform your Codeforces experience. It replaces the classic, dated Codeforces problem interface with a modern, highly optimized workspace built for competitive programmers.

## 🚀 Features

- **Integrated IDE**: A full Monaco editor (the same engine powering VS Code) embedded directly into the problem page.
- **Offline Reliability**: Your code, layouts, and editor state are automatically saved to local storage (`chrome.storage.local`). You'll never lose code due to an accidental refresh or tab close.
- **Smart Test Case Execution**: One-click "Run Samples" to test your code against all provided sample test cases directly in your browser without waiting in the server queue. 
- **Failed Test Case Integration**: Automatically fetches failed test cases from submission pages and allows you to instantly add them as custom sample tests for debugging.
- **Vim & Emacs Bindings**: Native support for Vim and Emacs keyboard shortcuts for power users.
- **Code Templates**: Save your favorite competitive programming boilerplate and load it instantly for any language with a single click.
- **Local-First Architecture**: Your data is yours. Everything stays locally on your device; no cloud sync required.

## 🛠️ Tech Stack

- **React 19**
- **Vite** & `@crxjs/vite-plugin` (for modern extension bundling)
- **Monaco Editor**
- **TypeScript**

## 📦 Installation & Usage

1. **Clone or Download** this repository.
2. Ensure you have Node.js installed, then run:
   ```bash
   npm install
   ```
3. Build the extension:
   ```bash
   npm run build
   ```
4. Load it into Chrome:
   - Navigate to `chrome://extensions/`
   - Turn on **Developer mode** in the top right.
   - Click **Load unpacked** and select the `dist` folder generated by the build step.
5. Open any Codeforces problem page (e.g., `codeforces.com/problemset/problem/1/A`), and the workspace will automatically launch!

## 🔧 Building for Production

If you make modifications to the extension, simply run `npm run build` again and refresh the extension in the `chrome://extensions/` page.
